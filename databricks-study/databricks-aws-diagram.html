üìù Key Architecture Patterns

‚úÖ What You CAN Do

  • Multiple AWS accounts linked to one Databricks account
  • Multiple regions per AWS account
  • Multiple workspaces per region
  • Multiple catalogs, schemas, tables per metastore
  • Share ONE metastore across multiple workspaces in same region

‚ùå What You CANNOT Do

  • Have more than ONE metastore per region
  • Share a metastore across different regions
  • Create workspace without a region
  • Use same metastore in us-east-1 and us-west-2

üéØ Best Practices

  • Plan metastore strategy carefully (it's the hardest to change)
  • Use workspace isolation for security boundaries
  • Organize catalogs by environment or domain
  • Consider AWS account separation for billing
  • Design for multi-region from the start if needed
